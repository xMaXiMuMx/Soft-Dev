# ประวัติของ Git </p>
Git คือ Version Control ตัวหนึ่ง ซึ่งเป็นระบบที่มีหน้าที่ในการจัดเก็บการเปลี่ยนแปลงของไฟล์ในโปรเจ็คเรา มีการ backup code ให้เรา สามารถที่จะเรียกดูหรือย้อนกลับไปดูเวอร์ชั่นต่างๆของโปรเจ็คที่ใด เวลาใดก็ได้ หรือแม้แต่ดูว่าไฟล์นั้นๆใครเป็นคนเพิ่มหรือแก้ไข หรือว่าจะดูว่าไฟล์นั้นๆถูกเขียนโดยใครบ้างก็สามารถทำได้ ฉะนั้น Version Control ก็เหมาะอย่างยิ่งสำหรับนักพัฒนาไม่ว่าจะเป็นคนเดียวโดยเฉพาะอย่างยิ่งจะมีประสิทธิภาพมากหากเป็นการพัฒนาเป็นทีม </p>
# คำสั่งพื้นฐานที่ใช้ใน Git </p>
- cd คำสั่ง Change Directory เป็นคำสั่งที่ใช้ในการเปลี่ยน directory </p>
- ls คำสั่ง list เป็นที่ใช้สำหรับแสดงแฟ้มข้อมูล รูปแบบคำสั่ง ls option ที่มักใช้กันใน ls คือ </p>
-l จะแสดงผลลัพธ์แบบ Long Format ซึ่งจะแสดง Permission ของแฟ้มด้วย </br>
-a จะแสดงแฟ้มข้อมูลทั้งหมด </br>
-F จะแสดง / หลัง Directory และ * หลังแฟ้มข้อมูลที่ execute ได้ </p>
- mkdir คำสั่ง make directory เป็นคำสั่งที่ใช้สำหรับการสร้าง directory </p>
- rmdir คำสั่ง remove directory เป็นคำสั่งที่ใช้สำหรับการลบ directory </p>
- echo  คำสั่งecho เป็นการให้แสดงข้อความ </p>
- mv คำสั่ง move เป็นคำสั่งที่ใช้สำหรับการย้ายแฟ้มข้อมูลและ Directory รวมถึงการเปลี่ยนชื่อด้วย </p>
- rm เป็นคำสั่งที่ใช้ลบไฟล์หรือ directory </p>
- git init เป็นคำสั่งที่ใช้สร้างแฟ้มข้อมูลย่อยชื่อ .git ที่เก็บแฟ้มโครงสร้างของ git repository ที่ยังไม่มีเนื้อหา </p>
- git add เป็นคำสั่งที่ใช้ stage เพื่อติดตามตามความเปลี่ยนแปลงของไฟล์ </p>
- git commit เป็นคำสั่งที่ใช้เพื่อบันทึกความเปลี่ยนแปลงที่เกิดขึ้นสู่ local repo </p>
- git push เป็นคำสั่งที่ใช้เพื่อส่ง commit ไปยัง remote repo </p>
- git clone เป็นคำสั่งที่ใช้เพื่อคัดลอก repo จาก remote มายัง local </p>
- git fetch เป็นคำสั่งที่ใช้ดึงความเปลี่ยนแปลงจาก remote มายัง local แต่ยังไม่รวมเข้าด้วยกัน </p>
- git merge เป็นคำสั่งที่ใช้รวมความเปลี่ยนแปลงที่ได้มาจาก fetch เข้ากับ local </p>
- git pull เป็นคำสั่งที่ใช้ดึงความเปลี่ยนแปลงจาก remote มายัง local และรวมเข้าด้วยกัน </p>
- git log เป็นคำสั่งที่ใช้เพื่อดูว่า git repo มี commit อะไรแล้วบ้าง </p>
- git status เป็นคำสั่งที่ใช้่เพื่อรตรวจสอบสถานะ
- git config --global user.email "email ที่เราใช้งาน" เป็นคำสั่งที่ใช้เพื่อกำหนดอีเมลล์ผู้ใช้ </p>
- git config --global user.name "username ที่เราใช้งาน" เป็นคำสั่งที่ใช้เพื่อกำหนดชื่อผู้ใช้  </p>
